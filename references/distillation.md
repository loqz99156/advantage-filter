# 方法蒸馏参考

当用户要求提炼高手方法、学习某套方法、反向抽象、复刻判断方式，或“像某人一样判断”时加载。

用途：把样本蒸馏成抽象判断结构，而不是模仿人物口吻或把具体人物当内部角色。

## 蒸馏步骤

1. 提取样本处理的问题类型。
2. 找出它反复使用的判断变量。
3. 抽象变量之间的因果关系。
4. 区分可迁移方法和不可迁移素材。
5. 把方法改写成可执行检查链。
6. 为每个检查点定义淘汰标准。

## 禁止事项

- 不把具体人物当成内部角色。
- 不复刻语气、权威姿态或个人标签。
- 不把案例当规则。
- 不把单一成功样本泛化成普遍真理。

## 用在个人优势定位中

从原始材料中抽象出的不是“赵丽颖/杨天真式人设”，而是：

```text
真实素材 -> 反差结构 -> 稳定特质 -> 外显物 -> 市场需求 -> 商业承接 -> 账号标签训练
```

当用户提供某个博主、名人、账号或文章作为参考时，只提炼其判断结构，不建议用户扮演那个人。
