# 抽象模型参考

当任务需要多个抽象模型，或需要用锋利内核模型删掉泛建议、标出测试假设和硬缺口时加载。

用途：选择最少但足够的模型组合，避免方法论堆砌。

## 核心模型

### 1. 定价权模型

```text
优势 = 稳定特质 × 可传播叙事 × 明确需求 × 商业承接
```

用于判断一个特质能不能成为主定位。

### 2. 信任路径模型

```text
陌生 -> 看懂你是谁 -> 相信你有证据 -> 感到你能帮我 -> 产生互动/私信/购买
```

用于判断内容是否能从曝光走到转化。

### 3. 内容资产模型

```text
故事母题 -> 固定栏目 -> 种子选题 -> 用户反馈 -> 产品/服务线索
```

用于把人设转成可持续内容系统。

### 4. 标签训练模型

```text
单一标签输入 -> 平台识别 -> 精准分发 -> 用户反馈 -> 标签强化 -> 副定位扩展
```

用于处理账号早期内容混乱。

### 5. 硬缺口模型

```text
没有证据 / 没有受众 / 没有内容形态 / 没有商业承接 / 没有验证数据
```

用于阻止过早定论。

## 使用规则

- 不要一次套所有模型。
- 每次只选择能解决当前卡点的 1-3 个模型。
- 模型必须改变输出：删掉一个泛建议、明确一个测试假设、标出一个硬缺口，或把结论变成动作。
